Here is a summary checklist of the commands and actions you need to perform to complete the **Getting started with Kubernetes** module:

### 1. Basic Commands and Imperative Pods

* **Action:** Create a pod imperatively.
```bash
kubectl run nginx --image=nginx

```


* **Action:** Verify pod status.
```bash
kubectl get pod

```



### 2. Declarative Configuration (YAML)

* **Action:** Create `wordpress.yml` and apply it.
```bash
kubectl apply -f wordpress.yml

```


* **Action:** Delete pods.
```bash
kubectl delete pod nginx wordpress

```



### 3. Deployments

* **Action:** Create a deployment imperatively with 10 replicas.
```bash
kubectl create deployment nginx-deployment --image=nginx --replicas=10

```


* **Action:** Create a deployment declaratively (`deployment.yml`).
```bash
kubectl apply -f deployment.yml

```


* **Action:** Test self-healing by deleting a pod.
```bash
kubectl delete pod <pod-name>
kubectl get pods # Watch it come back

```



### 4. Services (Networking)

* **Action:** Create a ClusterIP service imperatively.
```bash
kubectl expose deploy nginx-deployment --port=80 --type=ClusterIP

```


* **Action:** Test connectivity from inside the cluster.
```bash
kubectl run curl --image=curlimages/curl -i --tty -- sh
# Inside the pod:
curl <service-ip>

```


* **Action:** Expose service via NodePort.
```bash
kubectl expose deployment nginx-deployment --port=80 --type=NodePort

```


* **Action:** Create NodePort declaratively (`serviceNodePort.yml`) on port 30000.
```bash
kubectl apply -f serviceNodePort.yml

```



### 5. ReplicaSets and Namespaces

* **Action:** Scale a deployment.
```bash
kubectl scale deploy nginx-deployment --replicas=1

```


* **Action:** Create a namespace.
```bash
kubectl create namespace datascientest

```


* **Action:** Deploy resources into the new namespace.
```bash
kubectl create deployment wordpress --image=wordpress --replicas=3 --namespace datascientest

```


* **Action:** Expose the application in that namespace.
```bash
kubectl expose deployment wordpress --port=80 --type=NodePort --namespace datascientest

```
